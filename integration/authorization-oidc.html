Introducción

Para proporcionar integración entre los sistemas de gestión de usuarios del cliente (UMS) y Beacon, Brightcove ha implementado Open ID Connect (OIDC) para la forma más confiable de proporcionar autenticación.

Open ID Connect (OIDC) es una capa sobre el protocolo OAuth 2.0 para la autenticación externa para servicios como Auth0 y permite a los clientes verificar la identidad del usuario final en función de la autenticación realizada por un servidor de autorización.

Estas son las características destacadas de nuestra integración OIDC:

  • Se pueden utilizar bases de datos de terceros existentes para autenticar a los usuarios de Beacon.
    •   
  • Los usuarios de Beacon se registran, autentican y cambian las contraseñas a través de sus aplicaciones Beacon.
  • Soporte de aplicaciones Beacon en compras de aplicaciones.
    •  
  • Las aplicaciones existentes y las aplicaciones Beacon comparten la misma base de datos de autenticación.
  • Los nuevos espectadores pueden registrarse en el sistema de terceros.
  • Flujos de trabajo de emparejamiento de dispositivos (segundo dispositivo, ingrese el código en la pantalla para autenticar la aplicación de TV).
  • Inicios de sesión sociales si son compatibles con el proveedor de OIDC.
  • Se admite la limitación de simultaneidad de Beacon Stream.
  • El proveedor de OIDC puede ofrecer métodos de autenticación adicionales más allá del nombre de usuario y psw (multifactor, sin contraseña, emparejamiento de dispositivos, emparejamiento de cuentas de Google).

Implementación de un proveedor de autenticación OIDC

Estos son los pasos detallados a seguir en Brightcove Beacon para implementar el proveedor de autenticación OIDC:

  1. Ingrese a su cuenta de Brightcove Beacon.
  2. Haga clic en el botón de engranaje en la parte superior derecha.
  3. Haga clic en el Proveedores de autenticación en el menú de la izquierda.
  4. Añade el Auth Endpoint: account / social_login .
  5. Para el Tipo de autenticación campo, seleccione openid_connect.
  6. Complete los campos para cada una de las configuraciones de OpenID Connect:
    La mayor parte de la información para completar los campos de este formulario está en su Cuenta Auth0 .
    • A) URL base: Revise el valor de este campo con su representante de Brightcove.
    • B) Revocar URL de actualización: Establecer como oauth / revoke
    • C) URL de códigos de emparejamiento: Establecer como oauth / dispositivo / código
    • D) Tiene audiencia: Revise el valor de este campo con su representante de Brightcove.
    • MI) URL de información de usuario: Establecer como Información de usuario
    • F) URL del token de acceso: Establecer como oauth / token
    • GRAMO) URL de cierre de sesión: Establecer como v2 / cerrar sesión
    • H) Ubicación de Jwks: Es el Conjunto de claves web JSON ubicado en el Configuración avanzada> Endpoints pestaña de su aplicación en su Cuenta Auth0.
    • I) Actualizar URL del token: Establecer como oauth / token
    • J) URL de autorización: Establecer como autorizar
    • K) Ámbitos: Revise este campo con su representante de Brightcove.
  7. Haga clic en el Guardar ajustes botón.
