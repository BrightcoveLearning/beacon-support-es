Introducción

La información del lado del cliente se pasa de las aplicaciones a SSAI, y luego SSAI incluye estos datos en la etiqueta VAST del cliente, que luego permite a los clientes orientar anuncios en función de estos datos. El propósito de esto es aumentar el valor del inventario de anuncios de los clientes y, por lo tanto, aumentar los ingresos publicitarios para su negocio.

Puede configurar cómo se transmiten estos datos a su servidor de anuncios o proveedor de anuncios.

Beneficios

Orientación de anuncios mejorada

  • ID de sesión de usuario: generamos una ID de sesión en cada dispositivo cuando un usuario abre la aplicación. Esto le permite limitar la frecuencia de los anuncios a esa sesión, excluir anuncios de la competencia para que no se publiquen juntos y rotar los anuncios secuencialmente en esa sesión. Estas capacidades de orientación son necesarias para los anuncios programáticos y de venta directa.
  • Más datos para anuncios programáticos: Las aplicaciones móviles y las aplicaciones CTV no admiten cookies, que es el método dominante para la segmentación de anuncios basada en usuarios / dispositivos en la web. Al incluir parámetros como Device ID, puede transmitir esta valiosa información a los compradores de anuncios programáticos, que luego mejorarán sus capacidades de orientación de anuncios, lo que generará mayores CPM.

Compatibilidad con la división del inventario de anuncios

Para las aplicaciones que se ejecutan en Amazon, Roku y Samsung (en los EE. UU.), Debe realizar una división 70/30 de inventario / ingresos de anuncios con la plataforma. Los clientes de Beacon monetizan el 70% del inventario de anuncios y la plataforma obtiene el 30% restante del inventario de anuncios. Con esta orientación publicitaria mejorada, puede usar una etiqueta publicitaria en todas sus aplicaciones y pasar los parámetros del dispositivo a su servidor de anuncios. Dentro de su servidor de anuncios, implementarán la lógica empresarial que divide el inventario de anuncios por dispositivo. Las plataformas también requieren pasar LAT (orientación limitada de anuncios) para la división del inventario de anuncios, que es un parámetro que ahora enviamos a su servidor de anuncios.

Mejoras en la privacidad del usuario

  • LAT "Orientación de anuncios limitada": permite a los usuarios de las aplicaciones Beacon optar por no recibir orientación de anuncios según el comportamiento del usuario.
  • Los usuarios tienen la capacidad de restablecer la ID de dispositivo única que asignamos al dispositivo en cualquier momento.

Notas

  • Esta función funciona en todas las plataformas / dispositivos iOS, Apple TV, Android, Roku, Fire TV, Web y Smart TV.
  • La orientación de anuncios de Beacon Apps se puede utilizar tanto con VOD SSAI como con Live SSAI.

Implementación

A continuación se muestra una lista de macros de valor que se pasará a la aplicación, que luego sustituirá los valores reales antes de enviarla al servidor de anuncios. Ellos son NO configurable.

Macros de valor
Macro Descripción
PLATAFORMA La plataforma de baliza adecuada: iOS, AppleTV, Android, Roku, Web, STV
SO SO completo (incluida la versión) - ejemplo iOS 14
MFGR el fabricante del dispositivo; (Web - navegador del usuario)
MODELO Modelo de dispositivo; (Web: versión del navegador)
APP_ID ID de la aplicación (por ejemplo, ID de paquete); Web: dominio de la aplicación
LUGAR Código de idioma ISO 639-1 (2 letras) seguido de un guión bajo y un código de país ISO 3166-1 (2 letras) (ejemplo en_us): idioma configurado en la aplicación
RDID ID de dispositivo reiniciable (tifa: Samsung, rida: Roku, adid: Android, etc. - ver Identificadores de dispositivo a continuación para obtener más valores) - si segmentación de anuncios limitada está configurado en la aplicación, entonces la identificación debe ser todo ceros (lo que significa que no es posible identificar al usuario / dispositivo)
TIPO DE IDENTIFICACIÓN Escriba ID reiniciable (tifa: Samsung, rida: Roku, adid: Android, etc. - ver Identificadores de dispositivo a continuación para obtener más valores)
IS_LAT Marca que indica si el usuario selecciona la segmentación de anuncios limitada: 0 si el usuario no ha optado por limitar la segmentación, 1 si limita la segmentación de anuncios
SID Identificador de sesión único (se restablece cada vez que se inicia la aplicación). No depende del inicio de sesión / registro del usuario. El cambio de estado de primer plano a segundo plano no cambia esta ID.

Pasos de VOD

Pieza de Video Cloud - VOD

  1. Cree una configuración de anuncios desde Página de configuración de anuncios del lado del servidor en Studio:
    1. Hacer clic Crear configuración de anuncios y seleccione VOD
    2. Agregue un nombre para la configuración de anuncios <./ li>
    3. Seleccione la respuesta de su anuncio: se recomienda VMAP o Vast. Si usa Vast, deberá agregar puntos de referencia a los videos para especificar dónde deben aparecer las pausas publicitarias.
    4. Pegue su etiqueta de anuncio (de su proveedor de anuncios).
    5. Copie su ID de configuración de anuncios, ya que lo necesitará para la configuración de publicidad de Beacon.

Pieza de baliza - VOD

  1. Abra la interfaz de usuario de Beacon Classic y vaya a la pestaña Publicidad:
    1. Hacer clic Agregar nueva configuración
    2. Nombre la configuración de anuncios
    3. Seleccione Brightcove SSAI
    4. Pegar en el ID de configuración de anuncio SSAI dentro ID de configuración de anuncios de VOD
  2. Una vez que haya configurado su configuración de anuncios en Studio y Beacon Classic, debe implementar los parámetros del lado del cliente que se pasarán al servidor de anuncios de los clientes. Los parámetros del lado del cliente se pueden pasar en las API de reproducción (proporcionadas en el punto de entrada VMAP / Manifest); estos estarán disponibles para la infraestructura SSAI en forma de {{url.xxx}} macros donde xxx corresponde a la clave de parámetro que se envía en la URL de la API de reproducción.
    1. Ingrese parámetros de orientación de anuncios en el Objetivo de anuncio campo en Beacon. La entrada será una cadena en forma de consulta. Ejemplo: d ={PLATFORM} os ={OS} man ={MFGR} mo ={MODEL} appid ={APP_ID} lan ={LOCALE} did ={RDID} dt ={IDTYPE} lat ={IS_LAT} sid ={SID}
    2. Puede configurar las claves antes del signo igual. Ellos DEBER coincidir con las claves que se agregan a la etiqueta de anuncio SSAI.
    3. A continuación, actualice su etiqueta de anuncio en el Configuración de SSAI Studio página. A continuación, se muestra un ejemplo de parámetros que se ingresaron en Beacon Classic Ad-Targeting y se agregaron a la etiqueta de anuncio SSAI: d ={PLATFORM} os ={OS} appid ={APP_ID} did ={RDID} dt ={IDTYPE} lat ={IS_LAT} sid ={SID}
    4. Estos parámetros se traducen a esto en esta etiqueta de anuncio: https://ads.brightcove.com/ads?tech=dfpadrules&dur=15&pre=3&d={{url.d}} os ={{url.os}} appid ={{url.appid}} did ={{url.did}} dt ={{url.dt}} lat ={{url.latl}} sid ={{url.sid}}

Pasos en vivo

  1. Primero, construya su cadena de parámetros de orientación de anuncios, ya que la necesitará en las configuraciones SSAI y Beacon. Puede reutilizar los parámetros para VOD o seleccionar un conjunto diferente.

    Ejemplo:

    ip ={{client.ipaddress}} num ={{random.int32}} d ={{PLATFORM}} os ={{OS}} did ={{RDID}} dt ={{IDTYPE}} lat ={{IS_LAT}}
  2. Cree una configuración de anuncios a partir de la página de configuración de anuncios del lado del servidor en Studio:
    1. Hacer clic Crear configuración de anuncios y seleccione Vivir
    2. Agregue un nombre para la configuración de anuncios
    3. Seleccione Vasto su respuesta al anuncio
    4. Pegue su etiqueta de anuncio, agregando su cadena de consulta de parámetros de orientación de anuncios.

      Ejemplo:

      https://ad-provider-host.com/path/to/ad-handler?ip={{client.ipaddress}} num ={{random.int32}} d ={{PLATFORM}} os ={{OS}} did ={{RDID}} dt ={{IDTYPE}} lat ={{IS_LAT}}
    5. Copie su ID de configuración de anuncios para las configuraciones de Beacon Advertising
  3. Abra la interfaz de usuario de Beacon Classic y vaya a la pestaña Publicidad
    1. Hacer clic Agregar nueva configuración
    2. Nombre la configuración de anuncios
    3. Seleccione Brightcove SSAI
    4. Pegue el ID de configuración de anuncios SSAI en ID de configuración de anuncios en vivo
  4. Pegue su cadena de parámetros de orientación de anuncios en el Objetivo de anuncio campo.

    Ejemplo:

    ip ={{client.ipaddress}} num ={{random.int32}} d ={{PLATFORM}} os ={{OS}} did ={{RDID}} dt ={{IDTYPE}} lat ={{IS_LAT}}
