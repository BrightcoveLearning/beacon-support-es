Fases de implementación

En este tema, examinará las fases de una implementación de Brightcove Beacon.

Introducción

En este documento, se proporciona una mirada de alto nivel a todo el proceso de implementación de Brightcove Beacon. Enfatizará que usted, como cliente, tiene una serie de tareas que realizar antes de reunirse con un representante de Brightcove para crear las aplicaciones de su dispositivo.

Descripción general

    En esta sección, se introducen las fases, se definen brevemente y se indica el resultado de la fase. Más adelante en el documento, se proporcionan listas de verificación, cuando corresponda, para ayudarlo a prepararse para cada fase.

  1. Fase 1: Prepárese para la creación de experiencias de Brightcove Beacon

    Más adelante en el proceso de implementación de Brightcove Beacon (Fase 4), creará Brightcove Beacon Experience. Hará esto mientras trabaja con un representante de Brightcove. En este momento necesitarás tener listos algunos artefactos de diseño, así como decisiones tomadas en las plataformas en las que deseas implementar, estrategia de monetización, estrategia publicitaria, etc.

    RESULTADO FINAL : Una colección de recursos y estrategias definidas para usar cuando se reúna con un representante de Brightcove para crear Brightcove Beacon Experience.

  2. Fase 2: Establecer relaciones con proveedores externos

    Existen numerosas relaciones con proveedores externos que debe establecer antes de reunirse con un representante de Brightcove para crear Brightcove Beacon Experience. Esto incluiría ponerse en contacto con los proveedores de las plataformas en las que elige implementar una aplicación, un proveedor de servidor de publicidad, Stripe para pagos en línea si lanza aplicaciones web y / o Smart TV, etc. Es MUY IMPORTANTE que este proceso comience lo antes posible para no causar retrasos en el momento del envío de la aplicación del dispositivo.

    RESULTADO FINAL : Cuentas creadas con todos los proveedores externos deseados para usar cuando se reúna con un representante de Brightcove para crear Brightcove Beacon Experience.

  3. Fase 3: Cree el diseño de su aplicación

    Aquí se realizan dos partes del diseño de su aplicación. La primera es planificar qué videos estarán disponibles para los clientes y cómo agruparlos y ordenarlos. En esencia, su arquitectura de información. La segunda parte consiste en elegir cómo deben verse las pantallas específicas de su aplicación, dadas una serie de opciones.

    RESULTADO FINAL : Dos hojas de cálculo, creadas cuando se trabaja con los documentos correspondientes, se completan y están listas para enviarse al personal de Brightcove cuando Brightcove Beacon Experience está lista para ser creada.

  4. Fase 4: Ingiera o agregue contenido y configure su experiencia Brightcove Beacon

    Aquí utilizará la Consola administrativa de Brightcove Beacon para configurar su experiencia. Aquí importará activos multimedia de Video Cloud y agregará otro contenido. También se realiza la configuración de monetización y maquetación. En este punto, no es necesario agregar TODO el contenido, pero se necesita al menos contenido representativo para realizar pruebas exhaustivas.

    RESULTADO FINAL : Se agrega contenido y se realiza la configuración a través de Brightcove Beacon.

  5. Fase 5: Cree aplicaciones de dispositivo a partir de su experiencia Brightcove Beacon

    Trabajará con un representante de Brightcove para crear Brightcove Beacon Experience. El trabajo realizado en las Fases 1-4 se utiliza para crear la experiencia. Desde Brightcove Beacon Experience, se crean las aplicaciones del dispositivo. En este punto, también se realizan las pruebas de aceptación del usuario para asegurarse de que las aplicaciones funcionen como se desea.

    RESULTADO FINAL : Los archivos de la aplicación del dispositivo se crean, prueban y están disponibles para su envío a los proveedores / tiendas deseados.

  6. Fase 6: Envíe las aplicaciones de su dispositivo a las tiendas correspondientes

    En este punto, tiene las aplicaciones para las plataformas deseadas en la mano, así como la configuración realizada, por lo que ahora puede enviar las aplicaciones.

    RESULTADO FINAL: Las aplicaciones del dispositivo se envían correctamente y los espectadores comienzan a usar las aplicaciones con la aprobación del proveedor.

Tiempo de las fases

Tenga en cuenta que las Fases 1 y 2 pueden y deben ejecutarse simultáneamente.

Por supuesto, una vez que se inician las aplicaciones del dispositivo, puede continuar configurando y agregando contenido a Brightcove Beacon sin tener que volver a enviar las aplicaciones de su dispositivo a sus respectivas tiendas.

Detalles - Fase 1: Prepárese para la creación de experiencias de Brightcove Beacon

Las siguientes son tareas para completar para la Fase 1:

Determine las plataformas de aplicaciones del dispositivo necesarias y las plataformas prioritarias
Determinar la estrategia de monetización
Determinar si se necesitan servicios de autenticación de usuarios personalizados
Obtenga los colores y logotipos de su marca para crear la experiencia y para las especificaciones del dispositivo
Asista a la capacitación de Video Cloud: seminario web o capacitación a pedido
Ingerir activos multimedia en Video Cloud; necesitarás un mínimo de 40 videos y crear 3 a 5 listas de reproducción que usan esos videos
Cree los campos personalizados necesarios en Video Cloud; ver el Campos personalizados de Video Cloud documento para más detalles
Determinar los requisitos de análisis
Determine las ubicaciones geográficas restringidas, si las hay

Detalles - Fase 2: Establecer relaciones con proveedores externos

Las siguientes son tareas para completar para la Fase 2:

Determinar la estrategia publicitaria y establecer una relación con el proveedor adecuado.
Establezca una relación con Stripe para pagos en línea, si inicia aplicaciones web o de dispositivos de Smart TV. Otras plataformas tienen un proceso de pago integrado.
Configure las cuentas de la tienda y confirme las credenciales. Los proveedores pueden incluir, entre otros:
  • Plataforma para desarrolladores de Apple
  • Plataforma para desarrolladores de Google
  • Plataforma para desarrolladores de Amazon
  • Plataforma para desarrolladores de Roku
  • Otras plataformas de desarrollo (por ejemplo, Samsung)
Ver el Envío de aplicaciones de dispositivos a las tiendas para obtener información y las URL correspondientes para iniciar este proceso.

Detalles - Fase 3: Cree el diseño de su aplicación

Debe pensar en cómo desea que las aplicaciones de su dispositivo se vean ante sus clientes. En esta fase debes trabajar con los dos documentos:

  • Creación del diseño de su aplicación: arquitectura de la información
  • Creación del diseño de su aplicación: opciones de diseño

Después de trabajar con estos dos documentos, tendrá dos hojas de cálculo, una creada en cada documento, que entregará al personal de Brightcove cuando sea el momento de crear Brightcove Beacon Experience (Fase 4). La información de las dos hojas de cálculo detalla los videos presentados a los usuarios, incluida la forma en que están agrupados y ordenados. También se seleccionan diseños de pantalla específicos.

Detalles - Fase 4: Ingiera o agregue contenido y configure su experiencia Brightcove Beacon

Es posible que haya realizado otras pruebas y configuraciones, pero en esta fase desea completar la configuración. No es necesario que se agregue TODO el contenido, pero para fines de prueba, asegúrese de tener contenido representativo para todos los tipos de medios que utilizará. Utilizará la Consola administrativa de Brightcove Beacon Experience para realizar más configuraciones.

Detalles - Fase 5: Cree aplicaciones de dispositivo a partir de su experiencia Brightcove Beacon

Esta fase se lleva a cabo con la ayuda del personal de Brightcove. El representante de Brightcove utiliza los activos, las relaciones con los proveedores, las estrategias definidas, etc. de las tres primeras fases para crear y realizar la configuración inicial de Brightcove Beacon Experience. A partir de esta colaboración, los archivos de la aplicación del dispositivo real se generan desde Brightcove Beacon. También en esta fase, puede realizar pruebas de aceptación en las aplicaciones del dispositivo.

Detalles - Fase 6: Envíe las aplicaciones de su dispositivo a las tiendas correspondientes

Para cada proveedor para el que tenga una aplicación de dispositivo, debe enviar la aplicación a la tienda correspondiente. En la Fase 2, previamente estableció una relación con la plataforma del proveedor, por lo que en esta fase envía la aplicación para comenzar el proceso de aprobación. En el Envío de aplicaciones de dispositivos a las tiendas documento, encontrará pasos específicos y enlaces a la documentación del proveedor correspondiente para ayudarlo en el proceso de envío.
